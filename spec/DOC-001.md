Migration guide (docs/migration-guide.md) must document every breaking change from API 31 through API 35 with: symptom description, affected API call or manifest element, Kotlin code example showing before/after fix, and the API level where the change takes effect.

## Acceptance Criteria

### AC-1: Every breaking change has required fields
- **Given** any breaking change section in `docs/migration-guide.md`
- **When** the section is reviewed
- **Then** it contains: (1) symptom (crash, silent failure, install block, etc.), (2) affected API/manifest element, (3) Kotlin before/after code example, (4) API level annotation

### AC-2: API 31 changes are complete
- **Given** the 11 documented API 31 breaking changes in CLAUDE.md
- **When** cross-referenced with migration-guide.md
- **Then** all 11 are present with full documentation

### AC-3: API 33 changes are complete
- **Given** the 7 documented API 33 breaking changes
- **When** cross-referenced with migration-guide.md
- **Then** all 7 are present with full documentation

### AC-4: API 34 changes are complete
- **Given** the 6 documented API 34 breaking changes
- **When** cross-referenced with migration-guide.md
- **Then** all 6 are present with full documentation

### AC-5: API 35 changes are complete
- **Given** the 8 documented API 35 breaking changes
- **When** cross-referenced with migration-guide.md
- **Then** all 8 are present with full documentation

### AC-6: Code examples are valid Kotlin
- **Given** any before/after code example in the guide
- **When** extracted and syntax-checked
- **Then** the Kotlin is syntactically valid and uses current API conventions
