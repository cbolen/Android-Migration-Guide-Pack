DataWedge intent API reference (docs/datawedge-intents-ref.md) must cover: profile creation and configuration via intent API, scan result receipt via broadcast receiver, SOFT_SCAN_TRIGGER usage, scanner input enable/disable, and profile switching — with intent action strings and extra keys for each operation.

## Acceptance Criteria

### AC-1: Profile creation documented
- **Given** a developer needs to create a DataWedge profile programmatically
- **When** they consult `docs/datawedge-intents-ref.md`
- **Then** they find the intent action (`com.symbol.datawedge.api.ACTION`), extra key for `CREATE_PROFILE`, and required parameters

### AC-2: Scan result receipt documented
- **Given** a developer needs to receive barcode scan data
- **When** they consult the reference
- **Then** they find the broadcast intent action, extra keys for scan data (`com.symbol.datawedge.data_string`, `com.symbol.datawedge.label_type`), and receiver registration pattern

### AC-3: SOFT_SCAN_TRIGGER documented
- **Given** a developer needs to trigger a scan programmatically
- **When** they consult the reference
- **Then** they find the intent action and extra for `SOFT_SCAN_TRIGGER` with `START_SCANNING`/`STOP_SCANNING`/`TOGGLE_SCANNING` values

### AC-4: Scanner enable/disable documented
- **Given** a developer needs to enable or disable the scanner input plugin
- **When** they consult the reference
- **Then** they find the `ENABLE_PLUGIN`/`DISABLE_PLUGIN` or `SCANNER_INPUT_PLUGIN` configuration extras

### AC-5: Profile switching documented
- **Given** a developer needs to switch between DataWedge profiles
- **When** they consult the reference
- **Then** they find the `SWITCH_TO_PROFILE` intent extra and its usage constraints
